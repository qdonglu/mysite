    # 谈谈JS的单线程，以及如何实现的异步调用（事件循环 Event loop）

    JS被设计成单线程的原因：浏览器是呈现视觉效果的，一方面是为了更快的渲染页面；另一方面，JS是可以操作DOM的，如果存在多线程同时操作DOM的情况，可能会引起冲突。

    事件循环：JS虽然是单线程的，但是其可以实现异步操作。首先明确以下知识点：

    • 宏任务：同步的代码(<script>)，setTimeout, setInterval，UI rendering

    • 微任务：promise, process.nextTick，MutationObserver，Object.observe(已废弃)

    • JS是单线程的，但是浏览器却是多线程的。除了解释和执行JS的主线程外，浏览器还有其他线程：

      • http请求线程
      • 事件触发线程
      • 定时器触发线程
      • 页面渲染线程

    • 主线程解析和执行JS代码时，遇到非同步的行为时，会将这类任务交给浏览器的其他线程。

    • 一次正确的事件循环如下：

      1. 执行同步代码，宏任务
      2. 执行栈为空，按顺序执行所有的微任务
      3. 如果有渲染任务，就渲染
      4. 进行下一轮循环

    # 闭包的理解，优缺点，使用场景

    闭包是一个可以访问其他函数内部的变量的函数。

    优点：闭包是JS的特性，可以让函数访问非自己作用域的变量

    缺点：滥用闭包会增加内存暂用，有可能出现内存泄漏

    使用场景：函数柯里化，防抖节流

    # 块级作用域，变量提升，let，const 与 var 的差异点

    • 块级作用域：ES6之前，JS有全局作用域和函数作用域，ES6之后，多了一个块级作用域，就是被{}包裹的变量，只在该作用域内生效
    • 变量提升：函数，类，变量，不论定义在哪里，都会被提升到作用域顶部
    • var的特点：var 定义的变量会将声明和赋值同时提升到全局作用域或函数作用域，没有块级作用域
    • let，const的特点：存在暂时性死区，不会变量提升，在声明前使用会报错；let 声明的变量可以修改，const声明的常量，不能修改。
