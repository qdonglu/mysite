    # CSS盒模型

    一个块级元素包含content，padding，border，margin。标准盒模型的宽高默认是content部分，IE盒模型的宽高则是包含border部分。可以通过box-sizing（content-box, border-box）改变盒模型。

    • CSS 隐藏元素的几种方式

      • display: none; 相当于从dom中移除了元素。不占位置，会引起重排和重绘。
      • visiblity: hidden; 元素还占据位置，不响应交互。
      • opacity: 0;元素还占据位置，可以响应交互。
      • position将元素移除到可视窗口外。

    # 浮动和清除浮动

    浮动元素会脱离文档流，元素改动不会导致上层元素重排，不会撑开父元素 清除浮动的方式：

    • 通过添加伪元素::after，最常用

      <style>
  .parent {
    width: 100px;
    background-color: #eee;
  }
  
  .parent::after {
    content:"";
    height:0;
    line-height:0;
    display:block;
    clear:both;
    visibility:hidden;
  }
  
  .child {
    float: right;
    width: 30px;
    height:30px;
  }
</style>

<div class="parent">
  <div class="child"></div>
</div>

    • 同级元素添加clear: both; ，清除浮动，撑开父元素

      <style>
  .parent {
    width: 100px;
    background-color: #eee;
  }
  
  .child {
    float: right;
    width: 30px;
    height:30px;
  }
  
  .clearfix {
    clear:both;
  }
</style>

<div class="parent">
  <div class="child"></div>
  <div class="clearfix"></div>
</div>

    • 父级元素同级元素添加clear:both;，清除浮动，不会撑开父元素

      <style>
  .parent {
    width: 100px;
    background-color: #eee;
  }
  
  .child {
    float: right;
    width: 30px;
    height:30px;
  }
  
  .clearfix {
    clear:both;
  }
</style>

<div class="parent">
  <div class="child"></div>
</div>
<div class="clearfix"></div>

    • 父级元素添加overflow: hidden;，清除浮动，撑开父元素

      <style>
  .parent {
    width: 100px;
    background-color: #eee;
    overflow: hidden;
  }
  
  .child {
    float: right;
    width: 30px;
    height:30px;
  }
</style>

<div class="parent">
  <div class="child"></div>
</div>

    # link 和 @import 引入CSS资源的区别

    <!-- link 方式 -->
<link rel='stylesheet' href='CSS文件' type='text/css' media='all' />
<!-- @import 方式 -->
<style type="text/css">
@import url('CSS文件');
</style>
    • link除了能加载CSS样式，还能加载其他资源；@import只能加载CSS样式
    • link加载的资源，会在页面加载时，一起加载；@import加载的样式，只有在页面加载完后才会加载
    • 兼容性，link是属于XHTML标签，不存在兼容问题；@import 是CSS2.1提出的，存在兼容问题

    # 外边距折叠（margin-collapse）

    两个或多个毗邻的普通流中的块元素垂直方向上的 margin 会折叠。

    发生折叠的外边距以大的为主 如何避免外边距塌陷：

    • 浮动元素，绝对定位元素，inline-block元素（可以避免兄弟块级元素的外边距塌陷）
    • 触发了BFC的元素（可以避免和其子元素的外边距塌陷）

    # rgba和opacity的区别：

    • rgba的透明只作用于当前元素的颜色或背景色，如果作用于背景色，并不会作用于元素内的内容
    • opacity的透明作用于整个元素及元素内的内容

    # CSS中的长度单位

    • px 像素单位，对于某一分辨率是固定的
    • em 相对长度单位，相对于当前元素的font-size， 1em = 16px
    • rem 相对长度单位，相对于根元素的font-size，1rem = 16px
    • vw，vh 相对长度单位，相当于视窗的宽高

    # BFC

    块级格式化上下文，指一个独立的渲染容器

    • 形成 BFC 的条件
      • 浮动元素，float 除 none 以外的值
      • 定位元素，position（absolute，fixed）
      • display 为以下其中之一的值 inline-block，table-cell，table-caption
      • overflow 除了 visible 以外的值（hidden，auto，scroll）
    • BFC的特点
      • 内部的 Box 会在垂直方向上一个接一个的放置。
      • 垂直方向上的距离由 margin 决定
      • bfc 的区域不会与 float 的元素区域重叠。
      • 计算 bfc 的高度时，浮动元素也参与计算
      • bfc 就是页面上的一个独立容器，容器里面的子元素不会影响外面元素。

    # 移动端1px问题，推荐办法: 媒体查询 + scale

    @media only screen and (-webkit-min-device-pixel-ratio: 2.0) {
  .border-bottom::after {
    transform: scale(0.5);
  }
}

@media only screen and (-webkit-min-device-pixel-ratio: 3.0) {
  .border-bottom::after {
    transform: scale(0.33);
  }
}
